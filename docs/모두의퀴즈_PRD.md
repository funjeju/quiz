## 1. 프로젝트 개요

### 1.1 서비스 비전
"지루한 뉴스를 재미있는 게임으로" — 매일 자동 수집된 뉴스를 AI가 퀴즈로 변환하여, 연령·성별·지역별 랭킹 대결을 통해 지식과 재미를 동시에 제공하는 **뉴스 기반 퀴즈 게임 플랫폼**.

### 1.2 핵심 가치
- 🗞️ **자동화**: Google RSS → AI 요약 → 퀴즈 자동 생성 파이프라인
- 🎮 **게임화**: 속도+정확도 기반 랭킹, 포인트, 시즌 대회
- 📱 **바이럴**: 카카오톡 공유 → 링크 클릭 즉시 퀴즈 플레이
- 🏆 **경쟁**: 연령/성별/지역별 리더보드, 왕중왕전

### 1.3 타겟 사용자
| 연령대 | 특성 | 주요 카테고리 |
|--------|------|--------------|
| 10대 | 학생, 트렌드 민감 | K팝, 연예, 스포츠 |
| 20대 | SNS 활발, 시사관심 | IT, AI, K팝, 국제 |
| 30대 | 뉴스 소비층 | 시사, 정치, 경제, IT |
| 40대 | 깊이 있는 정보 선호 | 정치, 시사, 국제, 인물 |
| 50대+ | 경험 기반 지식 | 시사, 정치, 인물, 여행 |

  - 스포츠 (sports)
  - K팝 (kpop)
  - AI (ai-tech)
  - 정치 (politics)
  - 여행 (travel)
  - 인물 (people)
  - IT (it)

    충북→충남→전북→전남→경북→경남→제주(고정)

#### 2.1.2 RSS 수집 규칙
- 수집 시각: 매일 **오전 6시** (KST) 자동 실행 (Vercel Cron)
- 수집 목표: 카테고리당 최소 **25개** 기사 (퀴즈 20개 생성 여유분)
- 대상 기간: 전날 00:00 ~ 23:59 기사
- 중복 제거: URL 해시 기반 dedup
- 저장: Firestore `news_articles` 컬렉션

### 2.2 AI 퀴즈 생성 시스템

#### 2.2.1 생성 파이프라인
```
[뉴스 기사 원문]
    ↓ Claude API (요약)
[200자 핵심 요약]
    ↓ Claude API (퀴즈 생성)
[연령대별 퀴즈 세트]
    ↓ Firestore 저장
[quiz_sets 컬렉션]
```

#### 2.2.2 퀴즈 유형
| 유형 | 설명 | 비율 |
|------|------|------|
| 객관식 4지선다 | 가장 기본, 모든 연령 | 60% |
| OX 퀴즈 | 빠른 판단, 모바일 최적 | 20% |
| 단답형 주관식 | 고난도, 성인층 | 10% |
| 이미지형 | 인물/장소 사진 첨부 | 7% |
| 영상연동형 | YouTube 클립 첨부 | 3% |

#### 2.2.3 연령대별 난이도 프롬프트 전략
```
10대: 쉬운 어휘, 흥미 위주, 최신 트렌드 연결, 유머 포함
20대: 중간 난이도, 배경지식 활용, 사회적 맥락 연결
30대: 심층 분석, 경제/사회적 영향, 전문 용어 허용
40대: 역사적 맥락, 비교 분석, 정책/제도 이해
50대+: 경험 기반, 심층 배경, 국제 비교, 통찰형
```

#### 2.2.4 퀴즈 생성 수량 규칙
- 카테고리당 기사 25개 수집 → 퀴즈 **20개** 생성 (연령대 공통 기반)
- 1개 기사 → 연령대별 변형 5세트 생성 (총 100개 퀴즈/카테고리)
- 데일리 퀴즈: 전 카테고리에서 **10문제** 큐레이션
- 카테고리 퀴즈: 카테고리당 **20문제**

### 2.3 퀴즈 게임 플레이

#### 2.3.1 게임 모드
| 모드 | 문제 수 | 제한시간 | 특징 |
|------|---------|---------|------|
| 데일리 퀴즈 | 10문제 | 문제당 20초 | 매일 1회, 전체 랭킹 |
| 카테고리 퀴즈 | 20문제 | 문제당 30초 | 카테고리별 랭킹 |
| 위클리 챌린지 | 30문제 | 문제당 25초 | 주간 누적 |
| 먼슬리 왕중왕 | 50문제 | 문제당 30초 | 월간 결선 |
| 이얼리 챔피언십 | 100문제 | 문제당 30초 | 연간 대회 |

#### 2.3.2 점수 산정 공식
```
문제 점수 = 기본점수(100) × 시간보너스 + 연속정답보너스

시간보너스:
  - 5초 이내: ×2.0
  - 10초 이내: ×1.5
  - 20초 이내: ×1.0
  - 시간초과: 0점

연속정답보너스(콤보):
  - 3연속: +50점
  - 5연속: +100점
  - 10연속: +300점

최종 점수 = Σ(문제점수) + 전체 만점 보너스(모두 맞췄을 때 +500점)
```

#### 2.3.3 랭킹 시스템
- **실시간 랭킹**: 당일 데일리 퀴즈 참여자 순위
- **필터 옵션**: 전체 / 연령대별 / 성별 / 지역별
- **리더보드 갱신**: 실시간 (Firestore onSnapshot)
- **뱃지 시스템**: 1위=금왕관, 2위=은왕관, 3위=동왕관, 10위내=엘리트

### 2.4 포인트 & 보상 시스템

### 2.6 어드민 대시보드

## 3. 비기능 요구사항

### 3.1 성능
- 퀴즈 로딩: 초기 렌더링 **1.5초 이내** (LCP)
- 랭킹 갱신: **실시간** (Firestore 리얼타임)
- RSS 수집 완료: 수집 시작 후 **10분 이내**
- AI 퀴즈 생성: 카테고리당 **3분 이내**

### 3.2 확장성
- 동시 접속: 초기 목표 1,000명 → 스케일업 가능 구조
- Firestore 샤딩 전략 적용
- Vercel Edge Functions 활용

### 3.3 보안
- Firebase Auth (Google OAuth 2.0)
- Firestore Security Rules (역할 기반)
- Admin API: Service Account + 환경변수
- Claude API Key: Vercel 환경변수 (클라이언트 비노출)

## 4. 사용자 플로우

### 4.1 신규 사용자
```
앱 접속 → 구글 로그인 → 프로필 설정(닉네임/성별/연령대/지역) → 
메인 대시보드 → 오늘의 데일리 퀴즈 → 결과 및 랭킹 확인 → 
카카오 공유 또는 카테고리 퀴즈 도전
```

### 4.2 재방문 사용자
```
앱 접속 (자동 로그인) → 데일리 퀴즈 알림 확인 → 
퀴즈 플레이 → 랭킹 확인 → 지난 퀴즈 복습 or 다른 카테고리 도전
```

### 4.3 카카오 공유 수신
```
카카오 링크 클릭 → 앱 접속 → 
[비로그인] 로그인 유도 팝업 → 로그인 → 해당 퀴즈 플레이
[로그인] 즉시 해당 퀴즈 플레이 → 공유자와 점수 비교
```

## 5. 배포 및 운영 계획

### 5.1 단계별 출시
| 단계 | 기간 | 내용 |
|------|------|------|
| Phase 1 (MVP) | 4주 | 뉴스 수집 + AI 퀴즈 + 데일리 퀴즈 + 기본 랭킹 |
| Phase 2 | 2주 | 카테고리 퀴즈 + 카카오 공유 + 포인트 시스템 |
| Phase 3 | 2주 | 위클리/먼슬리 대회 + 어드민 고도화 |
| Phase 4 | 4주 | 스폰서십 모델 + 보상 시스템 + SNS 로그인 추가 |

### 5.2 비즈니스 모델 (향후)
- **스폰서 배너**: 퀴즈 완료 후 스폰서 광고
- **스폰서 퀴즈**: 브랜드 협찬 퀴즈 세트
- **프리미엄 구독**: 광고 제거, 힌트, 상세 통계
- **대회 입상 보상**: 스폰서 경품, 포인트 현금화
